Counting sort works with by hashing or mapping an element to a index and counting the frequency of each number along the way. This algorithm works with a time complexity of O(N) but has a memory tradeoff for O(Max_Range) as it needs an additional array of the same range or a large hash. This sort is useful in scenarios where the range of the input values is not very large and this allows us to sort them efficiently.
